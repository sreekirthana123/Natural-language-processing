# 🤖 Human vs AI Text Classification

> A machine learning pipeline built during a **hackathon** to detect whether a piece of text was written by a **human** or generated by an **AI** system. The project progresses from classical ML models to a fine-tuned transformer — achieving up to **94.3% accuracy**.

---

## 📌 Project Overview

With the rise of AI-generated content, distinguishing human writing from AI output has become a critical challenge in content moderation, academic integrity, and research. This project tackles that problem using a full NLP pipeline — from raw text all the way to a fine-tuned DistilBERT transformer model.

**Task:** Binary Text Classification  
**Labels:** `0` = Human-written, `1` = AI-generated  
**College:** JBIET | B.Tech 2nd Year | 2024–25

## 🔬 Methodology

### 1. Data Cleaning
- Dropped all null values
- Stripped leading/trailing whitespace from text
- Removed duplicate rows
- Reset index

### 2. Feature Extraction (Classical Models)
- `TfidfVectorizer` with:
  - `ngram_range=(1, 2)` — unigrams and bigrams
  - `max_features=100,000`
  - `lowercase=True`

### 3. Logistic Regression
- Solver: `liblinear`
- `class_weight='balanced'`
- `max_iter=200`

### 4. Linear SVM
- `LinearSVC` with `C=1.0`
- `class_weight='balanced'`
- Outperformed Logistic Regression by **+1.4%**

### 5. DistilBERT Fine-Tuning
- Base model: `distilbert-base-uncased` (66M parameters)
- Tokenization: WordPiece, `max_length=128`, padded + truncated
- Classification head added on top (newly initialized)
- Trained using Hugging Face `Trainer` API
- **Best hyperparameters found:**
  - `learning_rate = 3e-5`
  - `per_device_train_batch_size = 16`
  - `num_train_epochs = 2`
  - `weight_decay = 0.01`

## 🧠 Why DistilBERT?

Traditional TF-IDF models treat each word independently — they miss the context and meaning behind word combinations. DistilBERT uses **self-attention** to understand the full context of every word in a sentence, making it significantly better at detecting subtle stylistic differences between human writing and AI-generated text.

DistilBERT is a **distilled (compressed)** version of BERT — 40% smaller and 60% faster, while retaining ~97% of BERT's performance. This makes it ideal for fine-tuning on classification tasks with limited compute.

---

## ⚠️ Notes

- The `UNEXPECTED` and `MISSING` key warnings during DistilBERT loading are **completely normal**. They appear because the pre-trained weights don't include the classification head — fine-tuning adds those new layers.
- TF-IDF + Linear SVM remains a strong option if **fast, lightweight inference** is needed without a GPU.
- The first run of the notebook used a **dummy 20-row dataset** for testing the pipeline — the real results use the full 100K dataset.
